# Interface principles

前台阅读页与管理后台遵循 `apple-design` skill，目标情绪是安静、可信、
可掌控。这里的 “Apple design” 指交互与排版原则，不是复制产品外观。

## Reading experience

- 系统字体优先；大标题收紧字距与行距，正文保持舒展行距。
- 每屏只有一个明确视觉重点，晨报内容优先于装饰。
- 浮动导航和工具层可使用克制的半透明材质，正文卡片不堆叠玻璃效果。
- 移动端优先保证单手浏览、足够触控面积和清晰返回路径。

## Interaction

- 按下时立即提供视觉反馈，操作完成后给出明确状态。
- 面板进入和退出沿相同路径，弹层从触发位置建立空间关系。
- 可直接操纵的元素才使用可中断弹簧；普通页面切换保持短促、无弹跳。
- 不在过渡过程中锁死输入，不使用无意义的循环动画。

## Accessibility

- `prefers-reduced-motion` 下用短淡入淡出替代位移和弹簧。
- `prefers-reduced-transparency` 下提高表面不透明度并移除模糊。
- `prefers-contrast: more` 下提供实色背景和明确边界。
- 状态不能只靠颜色表达；键盘焦点始终可见。

## Administration

- 直接使用具体标签，例如“重新生成晨报”，避免含糊的“执行”。
- 危险或不可逆操作需要确认；可逆筛选调整应允许撤销或再次修改。
- 密钥状态只显示“已配置/未配置”和环境变量引用，永不回显明文。
- 长任务展示当前阶段、完成状态与失败原因。
